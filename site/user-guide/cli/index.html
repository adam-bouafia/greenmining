    CLI Commands Reference¶

    Complete reference for all GreenMining command-line interface commands.

    Global Options¶

    These options apply to all commands:

    Option Description
    --help Show help message and exit
    --version Show version number

    Commands Overview¶

    Command Description
    fetch Fetch repositories from GitHub
    extract Extract commit history
    analyze Analyze commits for green patterns
    aggregate Aggregate and summarize results
    report Generate analysis report
    pipeline Run complete analysis pipeline
    status Show current analysis status
    analyze-url Analyze a single repository by URL
    analyze-urls Analyze multiple repositories by URL

    fetch¶

    Fetch repositories from GitHub based on search criteria.

    greenmining fetch [OPTIONS]

    Options¶

    Option Type Default Description
    --max-repos INTEGER 100 Maximum number of repositories to fetch
    --min-stars INTEGER 100 Minimum GitHub stars required
    --languages TEXT Python,Java,Go,... Comma-separated language list
    --keywords TEXT microservices Search keywords
    --token TEXT (from env) GitHub personal access token

    Examples¶

    # Fetch Python repositories about kubernetes
greenmining fetch --max-repos 50 --keywords "kubernetes" --languages "Python"

# Fetch popular microservices projects
greenmining fetch --max-repos 100 --min-stars 500

# Fetch with custom languages
greenmining fetch --languages "Python,Go,Rust" --keywords "cloud-native"

    Output¶

    Creates data/repositories.json:

    [
  {
    "name": "kubernetes",
    "full_name": "kubernetes/kubernetes",
    "url": "https://github.com/kubernetes/kubernetes",
    "stars": 98000,
    "language": "Go",
    "forks": 35000
  }
]

    extract¶

    Extract commit history from previously fetched repositories.

    greenmining extract [OPTIONS]

    Options¶

    Option Type Default Description
    --max-commits INTEGER 1000 Maximum commits per repository
    --exclude-merges FLAG true Exclude merge commits
    --exclude-bots FLAG true Exclude bot commits

    Examples¶

    # Extract up to 500 commits per repo
greenmining extract --max-commits 500

# Include merge commits
greenmining extract --no-exclude-merges

    Output¶

    Creates data/commits.json:

    [
  {
    "sha": "abc123def456",
    "message": "Optimize caching layer for better performance",
    "author": "developer",
    "date": "2024-01-15T10:30:00Z",
    "repository": "org/repo"
  }
]

    analyze¶

    Analyze extracted commits for green software patterns.

    greenmining analyze [OPTIONS]

    Options¶

    Option Type Default Description
    --batch-size INTEGER 10 Number of commits per batch
    --enable-diff-analysis FLAG false Enable code diff analysis (slower)

    Examples¶

    # Basic analysis
greenmining analyze

# With diff analysis (more accurate, slower)
greenmining analyze --enable-diff-analysis

# Custom batch size
greenmining analyze --batch-size 50

    Output¶

    Creates data/analysis_results.json:

    [
  {
    "sha": "abc123def456",
    "message": "Optimize caching layer",
    "green_aware": true,
    "patterns": ["Cache Static Data"],
    "category": "caching",
    "confidence": 0.85,
    "keywords_matched": ["cache", "optimize"]
  }
]

    aggregate¶

    Aggregate analysis results with statistical summaries.

    greenmining aggregate [OPTIONS]

    Options¶

    Option Type Default Description
    --enable-stats FLAG false Enable statistical analysis
    --enable-temporal FLAG false Enable temporal trend analysis
    --temporal-granularity CHOICE quarter day/week/month/quarter/year

    Examples¶

    # Basic aggregation
greenmining aggregate

# With temporal analysis by quarter
greenmining aggregate --enable-temporal --temporal-granularity quarter

# Full statistical analysis
greenmining aggregate --enable-stats --enable-temporal

    Output¶

    Creates data/aggregated_statistics.json:

    {
  "summary": {
    "total_commits": 5000,
    "green_aware_count": 1250,
    "green_aware_percentage": 25.0,
    "total_repos": 50
  },
  "pattern_distribution": {
    "Cache Static Data": 320,
    "Use Async Instead of Sync": 180
  },
  "temporal_analysis": {
    "periods": [...]
  }
}

    report¶

    Generate a comprehensive analysis report.

    greenmining report [OPTIONS]

    Options¶

    Option Type Default Description
    --format CHOICE markdown Output format (markdown/json)
    --output PATH data/green_analysis.md Output file path

    Examples¶

    # Generate Markdown report
greenmining report

# Custom output path
greenmining report --output reports/analysis.md

    Output¶

    Creates a Markdown report with:

    • Executive summary
    • Pattern distribution charts
    • Per-repository statistics
    • Temporal trends (if enabled)
    • Statistical analysis (if enabled)

    pipeline¶

    Run the complete analysis pipeline (fetch extract analyze aggregate report).

    greenmining pipeline [OPTIONS]

    Options¶

    Option Type Default Description
    --max-repos INTEGER 100 Maximum repositories
    --max-commits INTEGER 1000 Maximum commits per repo
    --keywords TEXT microservices Search keywords
    --min-stars INTEGER 100 Minimum stars

    Examples¶

    # Quick test pipeline
greenmining pipeline --max-repos 5 --max-commits 50

# Full pipeline for research
greenmining pipeline --max-repos 100 --max-commits 1000 --keywords "kubernetes"

    greenmining status

    analyze-url¶

    Analyze a single GitHub repository by URL using PyDriller.

    greenmining analyze-url URL [OPTIONS]

    Arguments¶

    Argument Description
    URL GitHub repository URL (HTTPS or SSH)

    Options¶

    Option Type Default Description
    --max-commits INTEGER 1000 Maximum commits to analyze
    --since DATE None Start date (YYYY-MM-DD)
    --to DATE None End date (YYYY-MM-DD)
    --clone-path PATH /tmp/greenmining_repos Clone directory
    --no-cleanup FLAG false Keep cloned repository
    --output PATH None Output JSON file

    Examples¶

    # Basic analysis
greenmining analyze-url https://github.com/pallets/flask

# Limit commits
greenmining analyze-url https://github.com/pallets/flask --max-commits 100

# Date range
greenmining analyze-url https://github.com/pallets/flask \
  --since 2024-01-01 --to 2024-06-30

# Save results
greenmining analyze-url https://github.com/pallets/flask \
  --output flask_analysis.json

    Supported URL Formats¶

    # HTTPS
https://github.com/owner/repo
https://github.com/owner/repo.git

# SSH
git@github.com:owner/repo.git

    analyze-urls¶

    Analyze multiple repositories by URL.

    greenmining analyze-urls [OPTIONS]

    Options¶

    Option Type Default Description
    --urls TEXT None Comma-separated URLs
    --file PATH None File with URLs (one per line)
    --max-commits INTEGER 1000 Max commits per repo
    --output-dir PATH data/ Output directory

    Examples¶

    # Multiple URLs inline
greenmining analyze-urls --urls "https://github.com/org/repo1,https://github.com/org/repo2"

# From file
echo "https://github.com/pallets/flask
https://github.com/django/django
https://github.com/fastapi/fastapi" > repos.txt

greenmining analyze-urls --file repos.txt --max-commits 200

    Exit Codes¶

    Code Meaning
    0 Success
    1 General error
    2 Configuration error
    3 GitHub API error
    4 File I/O error

    Environment Variables¶

    All CLI options can be set via environment variables:

    export GREENMINING_MAX_REPOS=50
export GREENMINING_MIN_STARS=200
export GREENMINING_KEYWORDS="cloud-native"
